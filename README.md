## 1. Введение

### Актуальность
Теплоэлектроцентрали (ТЭЦ) являются критически важными объектами инфраструктуры. Дефекты оборудования могут привести к серьезным авариям и экономическим потерям. Традиционные подходы к анализу дефектов на основе ручной обработки данных не позволяют оперативно выявлять тенденции и прогнозировать отказы.

### Цель работы
Разработка и развертывание ETL-конвейера для анализа дефектов оборудования ТЭЦ с использованием современных открытых технологий (Apache Airflow, ClickHouse, Apache Superset) и контейнеризации Docker.

### Задачи
1. Провести анализ предметной области
2. Выполнить обзор и сравнительный анализ технологий
3. Обосновать выбор технологического стека
4. Разработать архитектуру ETL-конвейера
5. Реализовать контейнеризацию компонентов
6. Разработать DAG в Apache Airflow
7. Настроить ClickHouse
8. Создать дашборды в Apache Superset

## 2. Теоретическая часть

### 2.1. ETL-процессы
ETL (Extract, Transform, Load) — процесс, включающий:
- **Extract** — извлечение данных из различных источников
- **Transform** — очистка, нормализация, обогащение данных
- **Load** — загрузка в целевое хранилище

### 2.2. Анализ предметной области
Выявлены следующие категории дефектов оборудования ТЭЦ:
- Течи, свечи, парение (93% от общего числа)
- Подшипники
- Датчики
- Сальники

### 2.3. Выбор технологического стека

| Компонент | Выбранная технология | Причина |
|-----------|---------------------|---------|
| Оркестрация | Apache Airflow | Стандарт индустрии, богатая экосистема |
| Хранилище | ClickHouse | Высокая скорость агрегаций, сжатие 10:1 |
| Визуализация | Apache Superset | Нативная поддержка ClickHouse, SQL Lab |
| Контейнеризация | Docker + Compose | Простота развертывания, воспроизводимость |

## 3. Архитектура решения
### 3.2. Потоки данных

| № | Откуда | Куда | Что передается | Периодичность |
|---|--------|------|----------------|---------------|
| 1 | Внешний CSV | PostgreSQL | Исходные данные | Разово |
| 2 | PostgreSQL | ClickHouse | Очищенные данные | По расписанию |
| 3 | ClickHouse | Superset | Данные для визуализации | Real-time |

## 4. Реализация

### 4.1. Контейнеризация (docker-compose.yml)

```yaml
services:
  postgres:
    image: postgres:13
    environment:
      POSTGRES_USER: airflow
      POSTGRES_PASSWORD: airflow

  clickhouse:
    image: clickhouse/clickhouse-server:23.8
    ports:
      - "8123:8123"

  webserver:
    image: apache/airflow:2.8.2
    ports:
      - "8080:8080"

  superset:
    image: apache/superset:3.1.3
    ports:
      - "8088:8088"
```
